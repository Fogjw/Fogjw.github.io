## 写新文章

### 方式 A：用命令创建

```bash
hexo new "文章标题"
```

执行后会在 `source/_posts/` 下生成一个 `.md` 文件，文件名就是标题的拼音/英文。

### 方式 B：手动创建

直接在 `source/_posts/` 目录下新建 `.md` 文件，文件名建议用英文或拼音（会影响网址）。

## 文章结构说明

每篇文章开头都有一个 **YAML 头**（`---` 之间的内容），用来告诉博客这篇文章的信息：

```markdown
---
title: 文章标题
date: 2026-07-17
tags:
  - 标签A
  - 标签B
categories:
  - 分类名
---
正文内容写在这里，用 Markdown 语法。
```

### 各部分含义

| 字段           | 必填 | 说明                               |
| -------------- | ---- | ---------------------------------- |
| `title`      | 是   | 文章标题，会显示在页面标题和链接上 |
| `date`       | 否   | 发布日期，不写则以文件创建时间为准 |
| `tags`       | 否   | 标签列表，每行一个标签             |
| `categories` | 否   | 分类（只写一个，不支持多级）       |

### Markdown 基础语法

```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体** *斜体*

- 列表项 1
- 列表项 2

1. 有序列表 1
2. 有序列表 2

[链接文字](https://example.com)

![图片描述](/img/图片名.png)

`行内代码`

```代码块语言名
代码内容
```

```

## 分类与标签

### 分类和标签的创建

**不需要提前创建。** 在文章 YAML 头里写了什么分类/标签，系统会自动创建，没有就自动消失。

```markdown
---
title: 示例文章
tags:
  - JavaScript
  - 前端
categories:
  - 技术笔记
---
```

### 修改标签

打开文章 `.md` 文件，直接改 YAML 头里的 `tags` 列表即可：

```markdown
tags:
  - 新标签A
  - 新标签B
```

### 修改分类

同样直接改 YAML 头：

```markdown
categories:
  - 新分类名
```

### 删除分类或标签

把 YAML 头里对应的标签/分类行删掉即可。如果所有文章都不再用那个分类/标签，它就会自动从页面消失。

**注意：** 分类和标签是跟着文章走的——没有文章引用的分类或标签不会显示。

### 查看现有标签

访问博客 `/tags/` 页面可看到所有标签。

### 查看现有分类

访问博客 `/categories/` 页面可看到所有分类。

## 更新文章

1. 打开 `source/_posts/` 下对应的 `.md` 文件
2. 修改内容（标题、正文、标签、分类都可以改）
3. 保存文件
4. 在 Git Bash 中执行：

```bash
hexo clean && hexo generate
```

5. 本地预览（可选）：

```bash
hexo server
# 访问 http://localhost:4000 查看效果
```

6. 部署上线：

```bash
hexo deploy
```

或者双击 `deploy.bat`。

---

## 删除文章

1. 在 `source/_posts/` 目录下找到要删除的 `.md` 文件
2. 直接删除文件
3. 执行以下命令重新生成并部署：

```bash
hexo clean && hexo generate && hexo deploy
```

或者双击 `deploy.bat`。

## 设为私密文章

Hexo 默认**没有内置的密码功能**。有以下几种方案：

### 方案一：草稿（推荐）

把文章放在 `source/_drafts/` 目录下（而不是 `_posts/`），文章就不会被发布：

```bash
# 创建草稿
hexo new draft "草稿标题"
```

草稿在本地预览时可以用 `hexo server --draft` 查看，但部署时不会上线。

### 方案二：插件加密

安装 hexo-blog-encrypt 插件：

```bash
npm install hexo-blog-encrypt --save
```

然后在文章 YAML 头中添加：

```markdown
---
title: 加密文章
password: 你的密码
---
```

访问时需要输入密码才能查看内容。

## 创建新页面

除了文章，还可以创建独立页面（如"关于"、"友链"等）：

```bash
hexo new page "页面名称"
```

会在 `source/` 下创建同名目录和 `index.md` 文件，编辑这个文件即可。

创建后需要在 `_config.butterfly.yml` 的 `menu` 中添加导航链接才会显示在顶栏。

## 预览与部署

### 本地预览

```bash
hexo clean          # 清理缓存
hexo generate       # 生成静态文件
hexo server         # 启动预览 → http://localhost:4000
```

### 部署上线

```bash
# 方式一：直接部署
hexo deploy

# 方式二：一键部署
双击 deploy.bat
```

部署后等待 1-3 分钟，访问 `https://fogjw.github.io` 查看线上效果。

## 常见命令速查

| 命令                       | 作用          |
| -------------------------- | ------------- |
| `hexo new "标题"`        | 新建文章      |
| `hexo new page "页面名"` | 新建页面      |
| `hexo new draft "标题"`  | 新建草稿      |
| `hexo clean`             | 清理缓存      |
| `hexo generate`          | 生成静态文件  |
| `hexo server`            | 启动本地预览  |
| `hexo deploy`            | 部署到 GitHub |
| `hexo server --draft`    | 预览包含草稿  |
